# TLS / SSL

## Общая информация

### SSL (Secure Sockets Layer)

* Протокол защищённых соединений
* Разработан в 1995 году компанией Netscape
* На данный момент устарел и не используется
* Название сохранилось в продуктах:

  * OpenSSL
  * LibreSSL
  * SSL-сертификаты

---

### TLS (Transport Layer Security)

* Современный протокол защиты данных
* Стандартизирован IETF
* Актуальные версии:

  * TLS 1.2
  * TLS 1.3

---

## Место TLS в модели OSI

+ TLS работает между уровнями и частично покрывает:

* **Представления** → шифрование данных
* **Сеансовый** → управление сессией
* **Транспортный** → передача данных

---

## Основные свойства TLS

+ TLS обеспечивает:

1. **Конфиденциальность (Privacy)** → шифрование
2. **Целостность (Integrity)** → защита от изменений
3. **Аутентификация (Authentication)** → проверка подлинности

---

## Типы шифрования

### Симметричное шифрование

**Схема:**
`Клиент ←→ Сервер (один общий ключ)`

**Особенности:**

* используется один и тот же ключ
* высокая скорость

**Алгоритмы:**

* AES
* ChaCha20-Poly1305
* 3DES (устаревший)

---

### Асимметричное шифрование

**Схема:**

* открытый ключ → доступен всем
* закрытый ключ → хранится на сервере

**Особенности:**

* медленнее симметричного
* используется для обмена ключами

**Алгоритмы:**

* RSA
* Diffie-Hellman (DH)
* Elliptic Curve Diffie-Hellman (ECDH)

---

## Гибридное шифрование в TLS

+ TLS использует комбинацию:

* асимметричное шифрование → для обмена ключами
* симметричное шифрование → для передачи данных

> Это позволяет совместить безопасность и производительность.

---

## Обмен ключами (упрощённо)

1. Клиент отправляет `ClientHello`
2. Сервер отвечает `ServerHello` и отправляет открытый ключ
3. Клиент генерирует симметричный ключ
4. Клиент шифрует его открытым ключом сервера
5. Сервер расшифровывает ключ своим закрытым ключом
6. Далее используется симметричное шифрование

---

## Недостатки RSA

* Нет **совершенной прямой секретности (Perfect Forward Secrecy)**
* Если скомпрометирован закрытый ключ сервера → можно расшифровать старый трафик

### Уязвимости:

* Атака миллиона сообщений
* ROBOT attack

> В TLS 1.3 использование RSA запрещено для обмена ключами

---

## Diffie-Hellman (DH)

+ Алгоритм безопасного обмена ключами по открытому каналу.

### Условия:

* `p` — большое простое число (≥ 1024 бит)
* `g` — первообразный корень по модулю `p`

> Позволяет сторонам получить общий секрет, не передавая его напрямую.

---

## Diffie-Hellman Ephemeral (DHE / ECDHE)

* временные (одноразовые) ключи для каждой сессии
* ключ не хранится после завершения соединения

---

## Совершенная прямая секретность (PFS)

**Определение:**
+ Даже если закрытый ключ сервера скомпрометирован, прошлые сессии остаются защищёнными.

> Достигается с помощью DHE / ECDHE

---

## Итог

* SSL устарел → используется TLS
* TLS = шифрование + целостность + аутентификация
* Используется гибридное шифрование
* RSA устарел для обмена ключами
* Современный стандарт → ECDHE + TLS 1.3
