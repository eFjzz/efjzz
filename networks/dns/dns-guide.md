# DNS Protocol

+ DNS работает по модели **клиент–сервер**.

```
Клиент  → (запрос) →  DNS сервер
Клиент  ← (ответ)  ←  DNS сервер
```
---

# Транспортный уровень:

| Параметр                | Значение |
| ----------------------- | -------- |
| Основной протокол       | UDP      |
| Альтернативный протокол | TCP      |
| Порт                    | 53       |

## **Особенности:**

+ **UDP** используется в большинстве DNS-запросов
+ **TCP** используется:
> при передаче больших ответов

> при взаимодействии DNS-серверов

---

# Режимы работы DNS

## 1. Итеративный режим:

+ Сервер отвечает только той информацией, которой располагает:

1. Если сервер **обслуживает доменную зону** — возвращает ответ.
2. Если **не обслуживает** — возвращает адрес DNS-сервера, который знает больше.

---

## 2. Рекурсивный режим

+ В этом режиме **DNS-сервер сам выполняет поиск**.

+ Он обращается к другим DNS-серверам, пока не получит окончательный ответ.

---

# DNS Resolver - Сервер разрешения имен

+ ***DNS resolver*** — сервер, который выполняет разрешение доменных имен.

## Способы получения DNS сервера:

### 1. От провайдера или организации

+ DNS сервер передается клиенту через **DHCP** вместе с другими сетевыми настройками.

### 2. Открытые DNS серверы

| Провайдер          | DNS сервер |
| ------------------ | ---------- |
| Google             | 8.8.8.8    |
| Yandex             | 77.88.8.8  |
| Yandex (резервный) | 77.88.8.7  |

---

# Кэширование DNS

## Схема работы:

```
DNS клиент → DNS Resolver → Кэш
                          ↓
                   Дерево DNS
```

## Кэширование:
+ уменьшает нагрузку на DNS серверы

+ ускоряет получение ответа

---

# Типы ответов DNS:

| Тип ответа                         | Описание                                                    |
| ---------------------------------- | ----------------------------------------------------------- |
| Авторитетный (Authoritative)       | Ответ получен от сервера, который обслуживает доменную зону |
| Неавторитетный (Non-Authoritative) | Ответ получен из кэша другого сервера                       |

## **Особенности:**

+ Авторитетный ответ:

> данные берутся из файлов зоны на сервере

+ Неавторитетный ответ:

> данные берутся из кэша

> могут быть устаревшими

---

# Формат пакета DNS

+ В отличие от HTTP, DNS использует **бинарный формат данных**.

## Структура DNS пакета:

| Часть                     | Описание                       |
| ------------------------- | ------------------------------ |
| Заголовок                 | содержит идентификатор и флаги |
| Количество запросов       | число DNS запросов             |
| Количество ответов        | число DNS ответов              |
| Авторитетные серверы      | список серверов зоны           |
| Дополнительная информация | дополнительные записи          |

---

## Структура сообщения DNS:

| Часть          | Описание                  |
| -------------- | ------------------------- |
| Заголовок      | фиксированный размер      |
| Тело сообщения | содержит запросы и ответы |

### Особенности:

+ строго определенный формат
+ данные передаются **в бинарном виде**

---

# Формат DNS ответа

| Поле         | Значение          |
| ------------ | ----------------- |
| Имя          | site.com          |
| Тип записи   | 1 (A запись)      |
| Класс        | 1 (IN — Internet) |
| TTL          | 90 секунд         |

| Данные       | 178.255.130.68    |

---

# Флаги DNS пакета

| Флаг   | Значение              |
| ------ | --------------------- |
| QR     | 0 — запрос, 1 — ответ |
| OPCODE | тип запроса           |
| AA     | авторитетный ответ    |
| TC     | пакет был обрезан     |
| RD     | запрос рекурсии       |
| RA     | рекурсия доступна     |
| Z      | зарезервировано       |
| RCODE  | статус ответа         |

### Основные значения OPCODE

| Код | Значение           |
| --- | ------------------ |
| 0   | стандартный запрос |

### Основные значения RCODE

| Код    | Значение |
| ------ | -------- |
| 0      | успешно  |
| другие | ошибка   |

---

# Надежность DNS:

+ Каждую DNS зону обслуживают **несколько серверов**.

### Требования:

+ минимум **2 сервера**
+ серверы располагаются **в разных сетях**

+ Если один сервер не отвечает:

> клиент → отправляет запрос другому серверу зоны

---

# Безопасность DNS

## Особенности

| Проблема              | Описание                          |
| --------------------- | --------------------------------- |
| Передача данных       | данные передаются в открытом виде |
| Ограничения протокола | возможны атаки                    |

### Пример атаки

* **DNS cache poisoning** (отравление DNS кэша)
