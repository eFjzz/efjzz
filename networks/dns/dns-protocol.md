# DNS Protocol

+ Людям неудобно работать с **IP-адресами**, поэтому используется **DNS** 

+ протокол, который преобразует **доменные имена компьютеров в IP-адреса**.

---

# Преимущества DNS

1. Понятные человеку имена (например `google.com` вместо `142.250.74.14`)
2. Возможность менять сетевую инфраструктуру без изменения имени сайта

---

# Утилиты в терминале Linux для работы с DNS:

| Утилита | Назначение |
|-------|-----------|
| `nslookup` | Получение информации о домене |
| `host` | Простой DNS-запрос к серверу |
| `dig` | Подробная диагностика DNS |

---

# Файл `/etc/hosts`

+ Файл `/etc/hosts` содержит **локальное сопоставление IP-адресов и имен компьютеров**.

### Пример:

`102.54.94.97 - server`
`38.25.63.10 - my-client`

## Недостатки:

1. Быстро увеличивается размер файла  
2. Сложно вносить изменения  
3. Возможны конфликты имен (много имен зарегистрировано на один IP)

---

# Особенности DNS

## Распределенная система

+ DNS работает как **распределенная база данных**:

+ нет единого сервера, где хранятся все имена
+ данные распределены между большим количеством серверов интернета

### Делегирование ответственности:

Пространство имен делится на **домены**, и за каждый домен отвечает отдельная организация.

## Надежность

Для обеспечения устойчивости используются:

- **дублирование DNS-серверов**
- **резервные сервера**

Это позволяет системе продолжать работу даже при отказе одного сервера

# Структура доменного имени

+ Пример доменного имени:

`www.site.com`

| Часть | Значение |
|------|---------|
| `www` | имя компьютера (хоста) |
| `site` | домен второго уровня |
| `com` | домен верхнего уровня |

---

# Корневой домен

+ Полное доменное имя включает **корневой домен**.

`www.site.com.`

> точка в конце означает **корень DNS-иерархии**.

---

# Дерево доменных имен

+ DNS организован в виде **иерархического дерева**.

| Домен | Назначение |
|------|-----------|
| `org` | некоммерческие организации |

# Инфраструктура DNS

+ Схема работы DNS:

`Клиент DNS → DNS Resolver → Дерево доменных имен`

| Компонент | Описание |
|-----------|---------|
| Клиент DNS | отправляет запрос на определение IP |
| DNS Resolver | выполняет поиск IP-адреса |
| DNS серверы | содержат информацию о доменах |

### DNS Resolver:

+ Resolver занимается **разрешением доменных имен**:

+ ищет IP-адрес

+ сначала проверяет **кэш**

+ если записи нет — обращается к DNS-серверам

---

# Распределение доменных имен:

## Регистраторы:

+ Регистраторы занимаются **распределением и регистрацией доменных имен**.

## Регистратор корневого уровня

**ICANN (Internet Corporation for Assigned Names and Numbers)** управляет:

+ корневой зоной DNS
+ глобальным распределением доменных имен

## Регистраторы зон первого уровня

+ Особенности:

+ требуется **аккредитация в ICANN**
+ может быть **один или несколько регистраторов**
+ они регистрируют **домены второго уровня**

---

# Регистрация домена

+ Регистрация доменного имени:

+ является **платной услугой**
+ доступна **организациям и физическим лицам**
