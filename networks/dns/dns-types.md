# DNS Record Types

+ DNS может выполнять различные задачи:

+ определять **IPv4 и IPv6 адреса** по доменному имени
+ задавать **несколько доменных имен для одного IP**
+ находить **почтовый сервер домена**
+ определять **DNS-серверы доменной зоны**
+ определять **доменное имя по IP-адресу**

---

# Resource Records (RR)

+ Каждая DNS запись называется **Resource Record (RR)** и имеет следующие параметры.

| Поле | Описание |
|-----|-----|
| Тип записи | определяет назначение записи |
| Класс записи | определяет тип сети |
| Значение | данные записи |

### Класс записи

| Класс | Значение |
|-----|-----|
| IN | Internet |

---

# Записи для IP-адресов

## A-запись

+ Запись **A** используется для определения **IPv4 адреса**.

| Домен | Тип | IP |
|-----|-----|-----|
| site.com | A | 178.255.130.68 |

---

## AAAA-запись

Запись **AAAA** используется для определения **IPv6 адреса**.

| Домен | Тип | IPv6 |
|-----|-----|-----|
| maps.google.com | AAAA | 2a02:6b8::242 |

### Пример запроса:

`nslookup -type=AAAA maps.google.com`

Пример ответа:

`Name: maps.google.com`
  
`Address: 2a02:6b8::242`


---

## CNAME Record

+ **CNAME (Canonical Name Record)** определяет **псевдоним для другого доменного имени**.

+ Пример:

`www.site.com`

Пример ответа:

`www.site.com`
 `canonical name = site.com`

### Альтернативный способ

+ Можно задать **несколько A-записей для одного IP-адреса**, но это менее удобно.

---

# MX Record

+ **MX (Mail Exchange)** запись определяет **почтовые серверы домена**.

+ Пример для Gmail:

| Приоритет | Почтовый сервер |
|-----|-----|
| 5 | gmail-smtp-in.google.com |
| 10 | alt1.gmail-smtp-in.l.google.com |
| 20 | alt2.gmail-smtp-in.l.google.com |
| 30 | alt3.gmail-smtp-in.l.google.com |
| 40 | alt4.gmail-smtp-in.l.google.com |

> Чем **меньше число**, тем **выше приоритет сервера**.

---

# NS Record:

+ **NS (Name Server)** запись указывает **DNS-серверы, которые отвечают за доменную зону**.

### Пример зоны:

+ Зона: `yandex.ru`

| DNS сервер | IP адрес |
|-----|-----|
| ns1.yandex.ru | 213.180.193.1 |
| ns2.yandex.ru | 93.158.134.1 |

### Особенность:

+ Вышестоящий домен использует **A-записи (glue records)**, чтобы определить IP-адреса серверов из NS-записей.

---

# Типы DNS зон:

## Прямая зона (Forward Zone):

+ Используется для определения **IP-адреса по доменному имени**.

`site.com → 178.255.130.68`

---

## Обратная зона (Reverse Zone):

+ Используется для определения **домена по IP-адресу**.

---

# Специальный домен для обратных зон:

+ Для reverse DNS используется специальный домен:

`in-addr.arpa`

### Особенности:

1. используется домен `in-addr.arpa`
2. IP-адрес записывается **в обратном порядке**

### Пример:

`77.88.55.88`
↓
`88.55.88.77.in-addr.arpa`

---

# PTR Record:

+ **PTR (Pointer)** запись используется в **обратной DNS зоне**.

> Она связывает **IP-адрес с доменным именем**.

### Пример:

`88.55.88.77.in-addr.arpa → yandex.ru`

---

# Другие типы DNS записей:

+ Существуют и другие, менее часто используемые записи DNS.

| Тип | Назначение |
|----|----|
| TXT | хранение текстовой информации |
| SRV | запись сервисов |
| SOA | информация о зоне |
