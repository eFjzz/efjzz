# Network Standardization

## Почему появилась стандартизация сетей

На раннем этапе развития компьютерных сетей **единых стандартов не существовало**.

Из-за этого оборудование и программы разных производителей **не могли работать друг с другом**.

Основные проблемы:

- несовместимость сетевого оборудования
- несовместимость программного обеспечения
- использование разных сетевых протоколов

Это сильно усложняло развитие сетей.

---

# Цель стандартизации

Главная цель стандартизации — обеспечить **совместную работу оборудования и программного обеспечения разных производителей**.

Благодаря стандартам сегодня:

- сетевое оборудование разных компаний может работать вместе
- операционные системы могут взаимодействовать по сети
- устройства используют одинаковые протоколы передачи данных

📌 Именно поэтому современные сети (например Интернет) могут объединять **миллиарды устройств по всему миру**.

---

# Типы сетевых стандартов

## 1. De jure (формальные стандарты)

**De jure** — это стандарты, принятые официальными организациями.

Особенности:

- закреплены юридически
- принимаются международными организациями
- разрабатываются через официальные процессы стандартизации

Пример:

- **ISO/OSI Model**

---

## 2. De facto (фактические стандарты)

**De facto** — это стандарты, которые стали популярными **естественным образом**.

Обычно это происходит так:

1. появляется новая технология
2. она получает широкое распространение
3. становится фактическим стандартом

Пример:

- **TCP/IP**

Изначально этот стек протоколов **не был формальным стандартом**, но стал основой Интернета.

---

# Организации, разрабатывающие сетевые стандарты

## ISO — International Organization for Standardization

**Международная организация по стандартизации.**

Разрабатывает различные международные стандарты.

Примеры:

- **OSI Model** — эталонная модель взаимодействия открытых систем
- **RFID** — технологии идентификационных карт (основа NFC)

---

## IEEE — Institute of Electrical and Electronics Engineers

**Институт инженеров по электротехнике и электронике.**

Одна из самых важных организаций в области сетевых технологий.

Разрабатывает стандарты **технологий передачи данных**.

---

# Основные стандарты IEEE

| Стандарт | Технология |
|--------|--------|
| IEEE 802.3 | Ethernet |
| IEEE 802.11 | Wi-Fi |
| IEEE 802.15.1 | Bluetooth |
| IEEE 802.15.4 | сети Интернета вещей (IoT) |
| IEEE 802.16 | WiMAX |

---

## IAB — Internet Architecture Board

**Совет по архитектуре Интернета.**

Отвечает за развитие архитектуры интернет-протоколов.

Включает несколько групп.

---

### IRTF — Internet Research Task Force

Группа исследователей Интернета.

Занимается:

- долгосрочными исследованиями
- перспективными сетевыми технологиями

---

### IETF — Internet Engineering Task Force

Группа инженеров Интернета.

Основные задачи:

- разработка интернет-протоколов
- публикация технических документов

---

# RFC документы

**RFC (Request for Comments)** — технические документы, описывающие работу интернет-протоколов.

Важно:

RFC **формально не всегда являются стандартами**, но фактически используются как спецификации протоколов.

---

# Примеры RFC стандартов

| RFC | Протокол |
|----|----|
| RFC 2616 | HTTP |
| RFC 793 | TCP |
| RFC 768 | UDP |
| RFC 791 | IP |
| RFC 792 | ICMP |
| RFC 826 | ARP |
| RFC 2131 | DHCP |

---

## W3C — World Wide Web Consortium

Консорциум, который разрабатывает **стандарты Web**.

Примеры:

- HTML
- CSS
- Web API
- другие веб-технологии

---

# Почему стандарты так важны

Стандарты позволяют работать вместе:

- оборудованию разных производителей
- программному обеспечению разных компаний
- различным платформам и операционным системам
- разным типам устройств

Без стандартизации **глобальная сеть Интернет была бы невозможна**.

Если нужно глубже понять работу сетей и протоколов, полезно читать официальные стандарты.

Основные источники:
