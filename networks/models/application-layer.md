# Application Layer (OSI / TCP-IP)

## Что такое прикладной уровень

**Прикладной уровень (Application Layer)** — это уровень сетевой модели, на котором работают **сетевые приложения**.

Он обеспечивает **взаимодействие программ на разных устройствах через сеть**.

Примеры сетевых приложений:

- Web-браузеры
- Почтовые клиенты
- Мессенджеры
- FTP-клиенты
- DNS-клиенты

---

# Хосты (Hosts)

**Host (хост)** — это сетевое устройство, на котором запускаются сетевые приложения.

+Примеры хостов:

- персональные компьютеры
- серверы
- смартфоны
- виртуальные машины

+ Главная роль хоста:

> запуск приложений, которые взаимодействуют друг с другом через сеть.

---

# Взаимодействие приложений

На **прикладном уровне** приложения взаимодействуют между собой **с помощью прикладных протоколов**.

+ прикладной уровень **изолирован от сетевого оборудования**
+ приложения **не знают**, как именно данные передаются по сети

> они просто отправляют данные через сетевые протоколы.

---

# Изоляция уровней

+ Протоколы прикладного уровня **не занимаются передачей данных по сети**.
+ Эти задачи выполняют **нижние уровни сети**.

---

## Связь с транспортным уровнем

Для передачи данных приложения используют **протоколы транспортного уровня TCP/IP**.

То есть:

Application Layer
↓
Transport Layer
↓
Internet Layer
↓
Network Access Layer


Для приложений создаётся иллюзия, что:

> программы на разных хостах обмениваются данными напрямую.

Хотя на самом деле данные проходят через множество сетевых устройств.

---

# Сервисы транспортного уровня

Транспортный уровень TCP/IP предоставляет **разные типы передачи данных**.

Основные протоколы:

- TCP
- UDP

---

# TCP (Transmission Control Protocol)

**TCP** — протокол надежной передачи данных.

Основные характеристики:

- устанавливает соединение между хостами
- обеспечивает **надежную передачу данных**
- гарантирует **порядок доставки**
- контролирует ошибки передачи
- передает **поток байтов**

TCP используется, когда важно:

- сохранить данные без потерь
- получить данные в правильном порядке

---

## Особенности TCP

+ TCP обеспечивает:

1. устанавливает сетевое соединение;
2. протокол управления передачей;
3. обеспечивает надежную передачу потока байтов - ( большая порция данных );

> поэтому TCP используется для передачи **больших объемов данных**.

---

## Примеры использования TCP

TCP используется в протоколах:

- HTTP
- HTTPS
- SMTP
- FTP
- SSH

---

# UDP (User Datagram Protocol)

**UDP** — протокол передачи данных без установления соединения.

Основные характеристики:

- не устанавливает соединение
- не гарантирует доставку
- не гарантирует порядок пакетов
- работает быстрее TCP

---

# Особенности UDP

UDP:

- не устанавливается соединение; 
- обеспечивает ненадежную передачу коротких сообщений;

UDP передает данные в виде **дейтаграмм (datagrams)**.

Это делает UDP **очень быстрым**, но **менее надежным**.

---

# Дейтаграмма

**Дейтаграмма** — это короткое независимое сообщение, передаваемое по сети.

> каждая дейтаграмма передается **отдельно и независимо от других**.

---

# Примеры использования UDP

UDP используется в приложениях, где важна **скорость передачи**, а не абсолютная надежность.

Примеры:

- DNS
- IPTV
- интернет-звонки
- онлайн-игры
- видеостриминг

---

# Сравнение TCP и UDP

| Характеристика | TCP | UDP |
|---|---|---|
| Тип передачи | соединение | без соединения |
| Надежность | высокая | низкая |
| Контроль ошибок | есть | нет |
| Порядок пакетов | гарантируется | не гарантируется |
| Скорость | ниже | выше |

---

# Итог

На прикладном уровне:

- работают сетевые приложения
- используется множество прикладных протоколов
- приложения взаимодействуют через транспортный уровень

Транспортный уровень предоставляет два типа сервисов:

- **TCP** — надежная передача данных
- **UDP** — быстрая передача без гарантий

Приложения работают так, будто **данные передаются напрямую между хостами**, хотя на самом деле используются все уровни сетевой модели.
