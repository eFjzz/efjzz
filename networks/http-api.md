# HTTP API

## Использование HTTP в API

**HTTP API (Application Programming Interface)** 

+ это способ взаимодействия программ через протокол HTTP.

### API позволяет:

> вызывать программы на сервере и получать ресурсы в ответе;

+ Ответ может быть в разных форматах:

+ JSON
+ XML
+ YAML
+ и др.

---

## Запрос к API

+ Пример: 

**определение региона по IP-адресу**

`http://ip-api.com/json/site.com`

### Структура URL:

| Часть | Значение |
|------|----------|
| `http://` | протокол |
| `ip-api.com` | адрес сервера |
| `/json` | формат ответа |
| `/site.com` | запрос |

---

## Ответ API

+ Пример ответа сервера:

json
`{
  "status": "success",
  "country": "Russia",
  "countryCode": "RU",
  "region": "MOW",
  "regionName": "Moscow",
  "city": "Moscow",
  "zip": "12345"
}`

> API возвращает данные в формате JSON.

---

## Передача параметров через URL:

+ **При вызове API часто нужно передавать параметры.**

+ Пример:

`http://ip-api.com/json/site.com?lang=ru&field=city`

## Структура запроса: 

| Часть        | Значение               |
| ------------ | ---------------------- |
| `?`          | начало параметров      |
| `lang=ru`    | язык ответа            |
| `field=city` | запрашиваемое поле     |
| `&`          | разделитель параметров |

> в данном примере мы запрашиваем ответ на русском языке.

## Примеры популярных API:

> Некоторые крупные сервисы предоставляют свои API:

1. YouTube Data API

2. HeadHunter API

3. VK API

4. Telegram API

### Пример API HeadHunter (hh.ru):

+ Поиск вакансий:

`https://api.hh.ru/vacancies?text=javascript`

> Поиск вакансий по ключевому слову javascript.

+ Поиск работодателей: 

`https://api.hh.ru/employers?type=agaming`

> Поиск работодателей по типу.

## HTTP методы в API:

+ API обычно используют HTTP методы, соответствующие операциям CRUD.

| HTTP Метод | Операция | Описание             |
| ---------- | -------- | -------------------- |
| GET        | Read     | получение данных     |
| POST       | Create   | создание данных      |
| PUT        | Update   | полная замена данных |
| PATCH      | Update   | частичное изменение  |
| DELETE     | Delete   | удаление данных      |

> технически можно использовать и GET для удаления, но это плохая практика.

## Сложности использования HTTP API:

+ При работе с API могут возникать сложности:

1. отсутствует единый стандарт разработки API

2. авторизация пользователей (часто используется OAuth)

3. поддержка разных версий API

4. сложные URL запросы

## Инструменты для работы с API:

+ Популярные инструменты для тестирования API:

| Инструмент | Назначение                                   |
| ---------- | -------------------------------------------- |
| Postman    | тестирование API через графический интерфейс |
| curl       | выполнение HTTP запросов через терминал      |

# Итог:

+ HTTP API позволяет программам взаимодействовать через интернет, используя:

1. HTTP запросы

2. URL параметры

3. различные форматы ответа (JSON, XML и др.)

> API активно используются во всех современных веб-сервисах.
