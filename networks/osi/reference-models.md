# Reference Model (Сетевая эталонная модель)

+ **Reference Model** — это абстрактная модель, которая описывает, как должны взаимодействовать сетевые системы и протоколы.

> **разделение сети на уровни для упрощения работы и понимания**

---

## Зачем нужна Reference Model:

+ стандартизация сетей
+ упрощение разработки протоколов
+ понимание, где возникает проблема
+ удобство обучения

---

## Основные модели:

| Модель | Описание |
|-------|--------|
| OSI | Теоретическая модель из 7 уровней |
| TCP/IP | Практическая модель из 4 уровней |

---

## Модель OSI:

| № | Уровень | Назначение |
|---|----------|-------------|
| 7 | Прикладной | Работа с приложениями (HTTP, HTTPS, DNS, FTP, SMTP, SSH) |
| 6 | Представления | Формат данных, шифрование (SSL/TLS) |
| 5 | Сеансовый | Управление сессиями (NetBIOS, RPC) |
| 4 | Транспортный | Доставка данных (UDP, TCP) |
| 3 | Сетевой | Маршрутизация и адресация (IP [IPv4, IPv6]) |
| 2 | Канальный | Передача данных (Ethernet, Wi-Fi [MAC]) |
| 1 | Физический | Передача битов (кабели, сигналы) |

---

## Модель TCP/IP:

| Уровень | Протоколы |
|----------|------------|
| Прикладной | HTTP, HTTPS, DNS, SMTP, FTP |
| Транспортный | TCP, UDP |
| Интернет | IP, ICMP |
| Сетевой доступ | Ethernet, Wi-Fi |
