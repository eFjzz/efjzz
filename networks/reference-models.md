# Network Reference Models

## Что такое эталонная модель сети

**Эталонная модель сети** — это модель, которая описывает:
- сервисы сети
- уровни взаимодействия сетевых компонентов

Она используется для **стандартизации и понимания работы сетей**.

---

# 1. OSI Model (ISO / OSI)

**Open Systems Interconnection (OSI) Reference Model**

Принята как стандарт в **1983 году**.

Модель включает **7 уровней** и используется как **универсальный язык для описания сетей**, хотя сама модель и её протоколы **почти не используются на практике**.

**Открытая система** — система, построенная на основе **открытых спецификаций**.

---

# 7 уровней OSI

| № | Уровень | Название |
|---|---|---|
| 7 | Прикладной | Application |
| 6 | Представления | Presentation |
| 5 | Сеансовый | Session |
| 4 | Транспортный | Transport |
| 3 | Сетевой | Network |
| 2 | Канальный | Data Link |
| 1 | Физический | Physical |

Ниже находится **среда передачи данных**.

---

# Назначение уровней OSI

## 1. Physical (Физический)

Отвечает за **передачу битов по физическому каналу связи**.

Задача:
- передача потока бит
- соблюдение скорости передачи

Физический уровень **не анализирует смысл передаваемых данных**.

---

## 2. Data Link (Канальный)

Обеспечивает **надёжную передачу данных между соседними устройствами**.

Задачи:

- установление логического соединения
- обнаружение и исправление ошибок
- согласование скоростей передачи

В широковещательных сетях (например **Wi-Fi**):

- управление доступом к среде передачи
- физическая адресация (**MAC-адреса**)

---

## 3. Network (Сетевой)

Отвечает за **объединение нескольких сетей в одну большую сеть**.

Задачи:

- логическая адресация (**IP-адреса**)
- выбор маршрута передачи пакетов
- согласование разных сетевых технологий

---

## 4. Transport (Транспортный)

Обеспечивает **передачу данных между процессами на разных устройствах**.

Особенности:

- определяет **уровень надёжности передачи**
- обеспечивает **правильный порядок сообщений**

OSI определяет **5 классов транспортного сервиса**:
- 0 — минимальный
- 4 — максимальный

Это **первый сквозной уровень**.

Это значит:

данные доставляются **от отправителя до получателя**, независимо от количества промежуточных устройств.

---

## 5. Session (Сеансовый)

Отвечает за **установку и управление сеансами связи**.

Задачи:

- управление диалогом (кто и когда передаёт данные)
- управление маркерами
- синхронизация передачи

Например: возможность **возобновить передачу после сбоя**.

---

## 6. Presentation (Представления)

Отвечает за **формат данных**.

Функции:

- преобразование форматов данных
- кодировка символов
- форматы чисел
- шифрование и дешифрование

Пример:
- **TLS**

---

## 7. Application (Прикладной)

Предоставляет **сетевые сервисы пользователю**.

Примеры:

- Web
- электронная почта
- мессенджеры
- доступ к файлам
- видео и аудио связь

---

# 2. TCP/IP Model

**TCP/IP** — модель, основанная на одноимённом **стеке протоколов**, который широко используется в интернете.

Она появилась при разработке сети **ARPANET** — предшественника современного интернета.

ARPANET объединяла сети с **разными технологиями**, поэтому потребовалась модель, которая позволяла бы их объединять.

Название происходит от двух основных протоколов:

- **TCP**
- **IP**

---

# Уровни модели TCP/IP

Модель включает **4 уровня**.

| Уровень | Название |
|---|---|
| 4 | Application |
| 3 | Transport |
| 2 | Internet |
| 1 | Network Access |

---

# Назначение уровней TCP/IP

## Network Access

Отвечает за взаимодействие с **сетевым оборудованием**.

Примеры технологий:

- Ethernet
- Wi-Fi

---

## Internet

Соответствует **сетевому уровню OSI**.

Функции:

- передача пакетов между сетями
- маршрутизация

Термины:

- **subnet** — подсеть  
- **internet** — объединённая сеть  
- **internetworking** — объединение сетей  

---

## Transport

Функции аналогичны транспортному уровню OSI.

Протоколы:

- **TCP** — надёжная передача
- **UDP** — быстрая передача без гарантий

---

## Application

В TCP/IP **сеансовый и уровень представления отсутствуют**, поэтому их функции выполняет **прикладной уровень**.

---

# Стек протоколов TCP/IP

## Application

- HTTP — работа с web
- DNS — преобразование имени в IP
- SMTP — передача электронной почты

---

## Transport

- TCP
- UDP

---

## Internet

- IP — передача пакетов
- ICMP — диагностика сети

---

## Network Access

- Ethernet
- Wi-Fi

---

# Инкапсуляция

В сетях используется **инкапсуляция**.

Это процесс, при котором:

**данные верхнего уровня помещаются внутрь данных нижнего уровня**.

---

# Примеры работы протоколов

### Работа с Web


HTTP → TCP → IP → Wi-Fi


---

### Запрос DNS


DNS → UDP → IP → Ethernet
