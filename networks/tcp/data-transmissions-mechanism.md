# Скользящее окно в TCP

## Подтверждение доставки

### Гарантия доставки данных в TCP

+ подтверждение доставки (ACK)
+ повторная отправка неподтвержденных сегментов

---

## Варианты подтверждения доставки:

| Метод | Уровень | Описание |
|-------|--------|----------|
| Остановка и ожидание | Канальный | Отправка одного сегмента и ожидание ACK |
| Скользящее окно | Транспортный | Передача нескольких сегментов без ожидания |

---

## Остановка и ожидание (Stop-and-Wait):

`Отправитель → Получатель`

`Данные →
← ACK
Данные →
← ACK`


| Особенность | Описание |
|-------------|----------|
| Простота | Легко реализуется |
| Недостаток | Низкая производительность |

---

## Скользящее окно (Sliding Window):

`Отправитель → Получатель`

`Данные →
Данные →
Данные →`

    ← Кумулятивное подтверждение (ACK)

| Особенность | Описание |
|-------------|----------|
| Параллельность | Передается несколько сегментов |
| Эффективность | Повышает скорость передачи |

---

## Время передачи:

+ Передача данных не является мгновенной.

| Параметр | Значение |
|----------|----------|
| Round-Trip Time (RTT) | 10 мс |
| Сегментов в секунду | 100 |
| Размер TCP сегмента | 1460 байт |
| Размер кадра (L2) | 1500 байт |
| Заголовок IP | 20 байт |
| Заголовок TCP | 20 байт |
| Скорость передачи | ~143 КБ/с |

---

## Принцип работы скользящего окна:

+ Поток байт:

`||||||||||||||||||||||||||||||||||||||||`

[ подтвержденные данные ][ отправленные ][ неотправленные ]

| Область | Описание |
|---------|----------|
| Подтвержденные данные | уже получены и подтверждены |
| Отправленные данные | отправлены, но без ACK |
| Неотправленные данные | еще не переданы |

---

## Размер окна:

+ Размер окна — это количество байт, которые можно отправить без получения подтверждения.

| Событие | Результат |
|--------|----------|
| Получен ACK | окно сдвигается вперед |
| Освобождается место | можно отправлять новые данные |

---

## Типы подтверждений в TCP:

### Кумулятивное подтверждение (ACK)

| Свойство | Описание |
|----------|----------|
| Принцип | подтверждает все байты до указанного |
| Поведение | ожидается следующий байт |
| Использование | по умолчанию |

---

### Выборочное подтверждение (SACK)

| Свойство | Описание |
|----------|----------|
| Принцип | подтверждает отдельные диапазоны байт |
| Эффективность | лучше при потерях |
| Особенность | расширение заголовка TCP |
