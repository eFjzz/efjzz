# Flow and congestion control in TCP

---

## Управление потоком (Flow Control)

* В сети могут использоваться устройства разной производительности
* Цель: предотвратить "затопление" медленного получателя быстрым отправителем

### Пример

```
Отправитель (ПК)  --->  Получатель (смартфон)
```

---

## Что происходит внутри хоста

```
Отправитель --> Данные --> Получатель
                 ↓
      Транспортная подсистема → Буфер → Приложение
```

* Данные сначала попадают в буфер
* Затем передаются приложению
* Важно избежать переполнения буфера

---

## Механизм управления

* Отправитель указывает **размер окна (Window)**
* Это количество байт, которое получатель готов принять

---

## Управление перегрузкой (Congestion Control)

* В сети множество устройств передают данные одновременно
* При высокой нагрузке возникает перегрузка

### Последствия

* Маршрутизаторы не успевают обрабатывать пакеты
* Пакеты начинают отбрасываться

---

## Коллапс перегрузки (Congestion Collapse)

* Произошел в **1986 году**
* Каналы были полностью загружены
* Скорость передачи данных резко упала (в ~100 раз)

---

## Решение проблемы

* Учет загрузки сети при передаче данных

### Подходы

| Подход             | Описание                           |
| ------------------ | ---------------------------------- |
| Фиксированное окно | Например, 8 сегментов              |
| Динамическое окно  | Меняется в зависимости от нагрузки |

---

## Окна в TCP

```
Отправитель --> Маршрутизаторы --> Получатель
   (Congestion Window)             (Flow Control Window)
```

* **Окно перегрузки (cwnd)** — ограничивает сеть
* **Окно получателя (rwnd)** — ограничивает буфер

---

## Влияние размера окна

### Маленькое окно

* Низкая загрузка сети
* Низкая скорость передачи

### Большое окно

* Перегрузка сети
* Потери пакетов
* Падение скорости

---

## AIMD (Additive Increase / Multiplicative Decrease)

* Основной алгоритм управления перегрузкой

```
w(t+1) = {
  w(t) + a   — если нет перегрузки
  w(t) × b   — если есть перегрузка
}
```

### Параметры

| Параметр | Значение                   |
| -------- | -------------------------- |
| a        | MSS (Maximum Segment Size) |
| b        | 1/2                        |

---

## Недостатки AIMD

* Медленный линейный рост окна

| Канал          | Оценка       |
| -------------- | ------------ |
| Медленные сети | Приемлемо    |
| Быстрые сети   | Неэффективно |

---

## Сигналы о перегрузке

* Потеря сегмента
* Увеличение задержки
* Сигналы от маршрутизаторов (ECN — Explicit Congestion Notification)

---

## Медленный старт (Slow Start)

* Альтернативный механизм

### Особенности

* Начальное окно: 2–4 сегмента
* Увеличение: удвоение при каждом ACK
* Рост: экспоненциальный

### При перегрузке

* Окно сбрасывается
* Процесс начинается заново
