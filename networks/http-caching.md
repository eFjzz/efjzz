# HTTP Кэширование:

**Кэширование** — это механизм сохранения ресурсов веб-страницы, чтобы не загружать их повторно с сервера.

## Какие ресурсы браузер может кэшировать:

| Тип ресурса | Примеры |
|---|---|
| Web-страницы | HTML |
| Изображения | JPG, PNG, SVG |
| Таблицы стилей | CSS |
| Скрипты | JavaScript |

### Преимущества кэширования:

| Преимущество | Описание |
|---|---|
| Быстрая загрузка | ресурсы берутся из локального кэша |
| Меньше сетевого трафика | меньше запросов к серверу |
| Меньше нагрузки на сервер | сервер реже отправляет файлы |

> поддержка кэширования **встроена в протокол HTTP**.

---

# Как браузер определяет, можно ли использовать кэш

## ***Заголовок Expires***

+ сервер может указать время, до которого ресурс считается актуальным.

### Пример:

`Expires: Sat, 15 Apr 2026 10:15:35 GMT`

### Назначение:

| Заголовок | Описание                                                       |
| --------- | -------------------------------------------------------------- |
| Expires   | указывает время, до которого можно использовать ресурс из кэша |

### Особенности:

+ не все серверы устанавливают этот заголовок

+ после указанного времени ресурс считается устаревшим

## ***Эвристики (Heuristics)***

+ если сервер не указал правила кэширования, браузер может принять решение самостоятельно.

### Пример:

+ если страница долго не менялась, браузер может показать копию из кэша.

## ***Заголовок Cache-Control***

+ Основной заголовок для управления кэшированием.

## Типы кэширования:

| Параметр | Значение                                      |
| -------- | --------------------------------------------- |
| no-store | ресурс нельзя сохранять                       |
| no-cache | можно сохранить, но нужно проверить у сервера |
| private  | хранить можно только в кэше браузера          |
| public   | можно хранить в разделяемом кэше              |
| max-age  | время хранения в секундах                     |

+ Пример использования:

`Cache-Control: private, max-age=600`

### Значения параметров:
| Параметр    | Описание                             |
| ----------- | ------------------------------------ |
| private     | хранить только в браузере            |
| max-age=600 | хранить ресурс 600 секунд (10 минут) |

### Conditional GET (условный GET):

+ Conditional GET — это HTTP-запрос, при котором сервер отправляет ресурс только если он изменился.

> Это позволяет:

1. уменьшить сетевой трафик

2. ускорить загрузку страниц

### Способы обнаружения изменений

1. ETag

+ ETag — это уникальный идентификатор версии ресурса.

> Обычно это хэш содержимого файла.

### Пример:

`ETag: "634af215-ed"`

2. По дате изменения

+ Сервер сообщает дату последнего изменения файла.

### Пример:

`Last-Modified: Sat, 15 Apr 2026 17:47:01 GMT`

---

## ***Conditional GET по ETag : ***

+ Браузер отправляет серверу запрос:

`If-None-Match: "634af215-ed"`

### Результат:

| Ситуация          | Ответ сервера               |
| ----------------- | --------------------------- |
| файл не изменился | 304 Not Modified            |
| файл изменился    | 200 OK и новая версия файла |

## Conditional GET по дате

+ Браузер отправляет заголовок:

`If-Modified-Since: Sat, 15 Apr 2026 17:47:01 GMT`

### Результат:

| Ситуация            | Ответ сервера    |
| ------------------- | ---------------- |
| ресурс не изменился | 304 Not Modified |
| ресурс изменился    | 200 OK           |

### Особенности:

+ ресурс не передается

+ браузер использует копию из кэша

### Возможные дополнительные заголовки

| Заголовок     |
| ------------- |
| Expires       |
| Last-Modified |
| Cache-Control |

## ***Прокси-сервер :***

+ Proxy Server — это сервер между пользователем и интернетом.

### Схема работы: 

> Пользователь → Proxy → Интернет

### Как работает кэш прокси:

+ пользователь запрашивает страницу

+ запрос проходит через proxy

+ страница сохраняется в разделяемом кэше

### Преимущество: 

+ Если другой пользователь запросит ту же страницу:

> она будет загружена из кэша proxy

> интернет-трафик уменьшится

## Реверсивный прокси-сервер:

+ Reverse Proxy находится ближе к веб-серверу, а не к пользователю.

### Схема:

> Пользователь → Reverse Proxy → Web Server

### Используется когда страницы:

+ не являются статическими

+ генерируются программами

## Принцип работы :

1. пользователь запрашивает страницу

2. сервер генерирует страницу

3. копия сохраняется в reverse proxy cache

## ***Если другой пользователь запрашивает ту же страницу :***

+ она берется из кэша

+ сервер не тратит время на повторную генерацию

### Преимущества:

| Преимущество              | Описание                 |
| ------------------------- | ------------------------ |
| Быстрая загрузка          | страницы берутся из кэша |
| Меньше нагрузки на сервер | меньше генерации страниц |
| Экономия ресурсов         | меньше CPU и памяти      |
