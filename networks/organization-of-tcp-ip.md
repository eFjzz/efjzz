# Organization of TCP/IP Networks

TCP/IP — это основная модель сетевого взаимодействия, используемая в Интернете.  
Она состоит из нескольких уровней, каждый из которых выполняет свою задачу.

Основные уровни:

1. Network Access (сетевые интерфейсы)
2. Internet (интернет-уровень)
3. Transport (транспортный уровень)
4. Application (прикладной уровень)

---

# 1. Network Access Layer (Уровень сетевых интерфейсов)

Этот уровень отвечает за **взаимодействие с физической сетью** и существующими сетевыми технологиями.

Он позволяет устройствам подключаться к сети через:

- Ethernet (проводная сеть)
- Wi-Fi (беспроводная сеть)

### Примеры сетевого оборудования

- **Wi-Fi Access Point** — точка доступа
- **Ethernet Switch** — коммутатор

В проводных сетях устройства обычно подключаются **к коммутатору (switch)**.

---

# Физическая адресация — MAC Address

Чтобы устройства могли передавать данные друг другу, каждое сетевое устройство имеет **MAC-адрес**.

MAC-адрес используется для:

- идентификации устройств в сети
- передачи данных внутри локальной сети

Он используется в технологиях:

- Ethernet
- Wi-Fi

MAC-адреса регламентируются стандартами **IEEE 802**.

---

## Формат MAC-адреса

Основные характеристики:

- длина: **6 байт (48 бит)**
- состоит из **шестнадцатеричных чисел**

### Примеры записи

IEEE формат:

1C-75-08-D2-49-45


IETF формат:


1C:75:08:D2:49:45


---

## Особенности MAC-адресов

1. Каждый сетевой адаптер имеет **встроенный MAC-адрес**
2. MAC-адреса должны быть **уникальными в локальной сети**
3. Если MAC-адреса совпадут, сеть **не сможет определить получателя**

---

# 2. Internet Layer (Интернет-уровень)

Этот уровень отвечает за **объединение разных сетей в одну большую сеть**.

Главная задача:

**найти маршрут передачи данных через сеть маршрутизаторов.**

---

## Сетевое оборудование

На этом уровне используется:

**Router (марштизатор)**

Он выполняет:

- соединение разных сетей
- маршрутизацию пакетов
- передачу данных между сетями

---

# Глобальная адресация

На интернет-уровне используется **глобальная адресация**.

Особенности:

- адрес **не зависит от технологии сети**
- используется для **уникальной идентификации устройства в сети**

---

# IP Address

**IP-адрес** — это глобальный адрес устройства в сети TCP/IP.

IP используется протоколом:

**IP (Internet Protocol)** — протокол межсетевого взаимодействия.

---

## Формат IPv4 адреса

IP-адрес состоит из:

- **4 чисел**
- диапазон каждого числа: **0–255**

Пример:


192.168.1.1


---

# Распределение IP-адресов

IP-адреса должны быть **уникальными во всем мире**.

Их распределением занимается международная система регистраторов.

---

## 1. IANA

**Internet Assigned Numbers Authority**

Главная организация, которая управляет распределением IP-адресов.

---

## 2. RIR (Regional Internet Registries)

Региональные регистраторы распределяют IP-адреса по регионам.

| Регион | Организация |
|------|------|
| Северная Америка | ARIN |
| Южная Америка | LACNIC |
| Африка | AFRINIC |
| Европа | RIPE NCC |
| Азия и Австралия | APNIC |

---

# 3. Transport Layer (Транспортный уровень)

Транспортный уровень отвечает за **передачу данных между приложениями**.

Он решает важную проблему:  
**как доставить данные конкретному приложению на устройстве.**

---

## Пример сетевого взаимодействия

Предположим есть клиент и сервер.


Client          Server

192.168.1.2     192.168.1.100

Browser ---------> Web Server

Процесс:

1. Браузер отправляет запрос серверу
2. Сервер отправляет ответ
3. Браузер отображает страницу пользователю

---

## Проблема

Если на компьютере открыто **несколько приложений**:

- браузер
- мессенджер
- почтовый клиент

сервер должен понять **какому приложению отправлять данные**.

Для этого используются **порты**.

---

# Port (Порт)

Порт — это **адрес приложения на транспортном уровне**.

---

## Основные характеристики портов

- диапазон: **1 – 65535**
- используется для идентификации **сетевого процесса**
- каждое приложение использует свой порт

На одном IP-адресе может работать **тысячи приложений**, потому что используются разные порты.

---

## Формат адреса

Сетевой адрес приложения записывается так:


IP:Port


Пример:


192.168.1.100:80


Это означает:

- IP — устройство
- Port — конкретное приложение

---

# 4. Application Layer (Прикладной уровень)

Прикладной уровень содержит **протоколы, с которыми работают приложения**.

---

## Основные протоколы

### HTTP
Протокол передачи **web-страниц**.

---

### HTTPS
Защищённая версия HTTP.

Использует:

- шифрование
- TLS

---

### DNS
**Domain Name System**

Используется для:

- преобразования доменного имени
- в IP-адрес

Пример:


google.com → 142.250.74.14


---

### SMTP
**Simple Mail Transfer Protocol**

Используется для:

- отправки электронной почты

---

# Сетевое оборудование и адресация

## Сетевое оборудование

| Уровень | Оборудование |
|------|------|
| Network Access | Wi-Fi Access Point, Ethernet Switch |
| Internet | Router |

---

## Типы адресации

| Уровень | Тип адреса |
|------|------|
| Network Access | MAC address |
| Internet | IP address |
| Transport | Port |

---

# Итог

Для передачи данных в сети используется **несколько уровней адресации**:


+ MAC address → определяет устройство в локальной сети
+ IP address → определяет устройство в глобальной сети
+ Port → определяет приложение на устройстве


Пример полного сетевого адреса:

192.168.1.100:80

Это означает:
- устройство: **192.168.1.100**
- приложение: **порт 80 (Web server)**
