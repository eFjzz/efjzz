# Transport Layer: Service, Protocol, Interface
---
## Основная терминология

+ Разделение service/protocol/interface — база архитектуры сетей

---
### Сервис (Service)

+ Сервис определяет, **что именно делает уровень**.

+ Это абстрактное описание возможностей:

* передача данных
* надежность
* порядок доставки

---

### Протокол (Protocol)

+ Протокол уровня N — это **набор правил и соглашений**, по которым взаимодействуют одинаковые уровни на разных устройствах.

+ Примеры:

* TCP
* UDP

---

### Интерфейс (Interface)

+ Интерфейс — это **набор операций**, которые нижний уровень предоставляет верхнему.

> Это то, через что разработчик взаимодействует с сетью.

---

## Почему важно разделение?

* **Сервис** → *что делает*
* **Протокол** → *как это реализовано*
* **Интерфейс** → *как этим пользоваться*

> Это ключевая концепция, которую любят на собеседованиях.

---

## Интерфейс транспортного уровня

### Сокеты (Sockets)

+ При разработке приложений ты **не работаешь напрямую с TCP/UDP**,
а используешь **сокет-интерфейс**.

+ Пример (Python):

```python
import socket

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)  # TCP
```

+ Через сокеты можно использовать:

* TCP
* UDP

---

## Сервисы транспортного уровня (TCP/IP)

### 1. Надежная передача потока байт

* Протокол: **TCP**
* Характеристики:

  * гарантированная доставка
  * сохранение порядка
  * контроль ошибок

---

### 2. Ненадежная передача сообщений

* Протокол: **UDP**
* Характеристики:

  * без гарантий доставки
  * быстрее TCP
  * используется для realtime (видео, игры)

---

## Типы передаваемых данных:

| Тип данных         | Надежность | Протокол |
| ------------------ | ---------- | -------- |
| Поток байт         | ✅ Да       | TCP      |
| Короткие сообщения | ❌ Нет      | UDP      |

---

## Чего не хватает в TCP/IP

+ Полезный сервис, которого **нет**:

### Надежная доставка коротких сообщений

* что-то среднее между TCP и UDP
* часто приходится реализовывать вручную

---

## Современные решения

### QUIC

* поддерживает:

  * надежную передачу
  * **несколько потоков одновременно**
* используется в HTTP/3
