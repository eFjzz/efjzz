# Установка соединения в TCP

## Гарантии TCP:

+ надежная доставка данных
+ сохранение порядка следования сообщений 

---

## Процесс передачи данных TCP:

| Этап | Описание |
|------|----------|
| 1 | Установка соединения  |
| 2 | Передача данных |
| 3 | Разрыв соединения |

---

## Задачи соединения:

| № | Задача |
|---|--------|
| 1 | Убедиться, что отправитель и получатель готовы к передаче данных |
| 2 | Синхронизация номеров потока байт |
| 3 | Договориться о параметрах соединения  |

---

## Установка соединения:

`Отправитель → Получатель`

`SYN, seq=7537 →
← SYN, seq=36829, ACK=7538
ACK=36830 →`


> Соединение установлено

| Шаг | Действие |
|-----|----------|
| 1 | Отправитель посылает SYN |
| 2 | Получатель отвечает SYN + ACK |
| 3 | Отправитель отправляет ACK |

---

## Зачем нужна синхронизация:

+ Несколько инкарнаций одного соединения:

| Шаг | Ситуация |
|-----|----------|
| 1 | Соединение установлено |
| 2 | Отправитель перезагрузился  |
| 3 | Создается новое соединение |

+ Проблема: одинаковые номера пакетов в разных инкарнациях соединения  

+ Решение: синхронизация предотвращает конфликты

---

## Разрыв соединения (Connection Termination)

### Грациозный разрыв (FIN)

`Отправитель → Получатель`

`FIN →
← ACK
← FIN
ACK →`

| Шаг | Действие |
|-----|----------|
| 1 | Одна сторона посылает FIN |
| 2 | Другая сторона подтверждает ACK |
| 3 | Вторая сторона посылает FIN |
| 4 | Финальный ACK отправлен |

---

### Сброс соединения (RST)

`Отправитель → Получатель`

`SYN →
← RST`

| Сценарий | Описание |
|----------|----------|
| RST | Мгновенный разрыв соединения |
