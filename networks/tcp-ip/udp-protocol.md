# UDP Protocol (User Datagram Protocol)
---
+ **UDP (User Datagram Protocol)** — протокол передачи данных без установления соединения транспортного уровня моделей **TCP/IP** и **OSI**.
+ Сообщение UDP называется **дейтаграммой**  
> (аналогия с телеграммой — короткое сообщение, доставляемое быстро).
> Надежность доставки не обеспечивается.
---
## Основные особенности UDP:
+ Нет установки соединения
+ Нет гарантии доставки данных
+ Нет гарантии порядка сообщений
+ Высокая скорость передачи
---
## Области применения UDP:

| Область | Описание |
|--------|--------|
| Клиент-сервер | Короткие взаимодействия (запрос–ответ) |
| Стриминг | Видео и голос (потери допустимы) |
| Онлайн-сервисы | Игры, DNS, VoIP |

---

## Формат заголовка UDP:

+ Порты используются для адресации на транспортном уровне.

| Поле | Размер |
|------|--------|
| Порт отправителя | 16 бит |
| Порт получателя | 16 бит |
| Длина UDP | 16 бит |
| Контрольная сумма | 16 бит |

---

## Длина UDP:

| Параметр | Значение |
|----------|---------|
| Минимум | 8 байт (только заголовок) |
| Максимум | 65 535 байт (ограничение IP-пакета) |

---

## Обработка ошибок:

+ UDP **не занимается обработкой ошибок**.

+ Если при передаче произошла ошибка:
+ обработка переносится на протоколы более высокого уровня

> например: **DNS**

---

## Использование UDP в DNS:

+ Пример работы:

| Клиент DNS | Сервер DNS |
|-----------|-----------|
| какой IP у site.com? | 178.248.237.68 |

+ Если ответ не пришёл:

+ клиент ждёт определённое время (timeout)
+ затем повторно отправляет запрос

---

# Особенности UDP

UDP:

- не устанавливается соединение; 
- обеспечивает ненадежную передачу коротких сообщений;

UDP передает данные в виде **дейтаграмм (datagrams)**.

Это делает UDP **очень быстрым**, но **менее надежным**.

---

# Дейтаграмма

**Дейтаграмма** — это короткое независимое сообщение, передаваемое по сети.

> каждая дейтаграмма передается **отдельно и независимо от других**.

---

# Примеры использования UDP

UDP используется в приложениях, где важна **скорость передачи**, а не абсолютная надежность.

Примеры:

- DNS
- IPTV
- интернет-звонки
- онлайн-игры
- видеостриминг

---

# Сравнение TCP и UDP

| Характеристика | TCP | UDP |
|---|---|---|
| Тип передачи | соединение | без соединения |
| Надежность | высокая | низкая |
| Контроль ошибок | есть | нет |
| Порядок пакетов | гарантируется | не гарантируется |
| Скорость | ниже | выше |

---

# Итог

На прикладном уровне:

- работают сетевые приложения
- используется множество прикладных протоколов
- приложения взаимодействуют через транспортный уровень

+ Транспортный уровень предоставляет два типа сервисов:

- **TCP** — надежная передача данных
- **UDP** — быстрая передача без гарантий

> Приложения работают так, будто **данные передаются напрямую между хостами**, хотя на самом деле используются все уровни сетевой модели.
