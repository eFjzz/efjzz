# TCP Protocol (Transmission Control Protocol)

+ протокол транспортного уровня, обеспечивающий надежную передачу данных.

---

## Сервис TCP:

+ надежная передача потока байт (reliable byte stream)

---

## Поток байт и сегменты:

+ Данные от приложения передаются как поток байт:

> Поток байт → Сегмент | Сегмент | Сегмент | Сегмент

| Этап | Описание |
|------|---------|
| 1 | Приложение передает поток байт |
| 2 | TCP разбивает поток на сегменты |
| 3 | Сегменты передаются по сети |
| 4 | Получатель собирает сегменты обратно в поток |

---

## Проблемы передачи в сети:

| Проблема | Описание |
|----------|----------|
| Потеря | сегмент может не дойти |
| Нарушение порядка | сегменты приходят не по порядку |
| Дублирование | сегменты могут повторяться |

---

## ✅ Гарантии TCP

| Гарантия | Описание |
|----------|----------|
| Доставка | данные будут доставлены |
| Порядок | данные будут собраны в правильном порядке |

---

## Подтверждение (ACK):

| Отправитель | Получатель |
|------------|------------|
| Сегмент данных → | |
| | ← ACK (подтверждение) |
| Сегмент данных → | |
| | ← ACK |

---

## Обработка ошибок:

`Сегмент данных → X (потерян)`

| Шаг | Действие |
|-----|----------|
| 1 | Отправитель устанавливает таймер |
| 2 | Если ACK не пришел — повторная отправка |
| 3 | Получение ACK после успешной доставки |

---
## 📊 Скользящее окно (Sliding Window)

| Параметр | Описание |
|----------|----------|
| Суть | подтверждаются сразу несколько сегментов |
| Цель | повышение производительности |

---

## Нумерация байт:

+ Сегменты: `[0–1023] | [1024–2047] | [2048–3071] | [3072–...]`

| Назначение | Описание |
|------------|----------|
| Порядок | восстановление последовательности |
| Контроль | обнаружение дубликатов |

---

## Пример работы ACK с нумерацией:

| Отправитель | Получатель |
|------------|------------|
| Сегмент (байт 0) → | |
| | ← ACK (ожидаю байт 1024) |
| Сегмент (байт 1024) → | |
| | ← ACK (ожидаю байт 2048) |

---

## Соединение TCP:

+ Перед передачей данных необходимо установить соединение.

### Задачи соединения:

| Задача | Описание |
|--------|----------|
| Согласование | обе стороны готовы к передаче |
| Синхронизация | согласование нумерации |
| Параметры | настройка соединения |

+ После завершения передачи:
> соединение разрывается
