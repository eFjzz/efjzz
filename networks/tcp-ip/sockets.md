# Интерфейс сокетов транспортного уровня TCP/IP

---

## Сокеты

* Сокет — это файл специального вида
* Все записанные в него данные передаются по сети
* Передача данных скрыта от программиста

### Особенность

* При изменении протоколов транспортного уровня программу менять не требуется

---

## Общая информация

* Сокеты — де-факто стандарт интерфейса транспортного уровня
* Впервые появились в **Berkeley UNIX 4.2 BSD (1983 г.)**
* Реализованы в разных ОС и языках программирования

---

## Сокеты Беркли (Berkeley Sockets)

| Операция    | Назначение                                  |
| ----------- | ------------------------------------------- |
| **socket**  | Создать новый сокет                         |
| **bind**    | Связать сокет с IP-адресом и портом         |
| **listen**  | Перевести сокет в режим ожидания соединений |
| **accept**  | Принять входящее соединение                 |
| **connect** | Установить соединение                       |
| **send**    | Отправить данные                            |
| **receive** | Получить данные                             |
| **close**   | Закрыть соединение                          |

---

## Модель клиент-сервер

### Сервер

* Работает на известном IP-адресе и порту
* Пассивно ожидает входящих соединений

### Клиент

* Активно устанавливает соединение
* Использует IP-адрес и порт сервера

---

## Работа сокетов

```
Сервер                Клиент
socket     ↔          socket
bind       ↔          connect
listen     ↔          send
accept     ↔          receive
receive    ↔          close
send
close
```
