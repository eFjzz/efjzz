# Протокол HTTP

## 1. HTTP это:
**HTTP (HyperText Transfer Protocol)** — протокол передачи гипертекста.  
Он является **основой World Wide Web** и используется для передачи веб-страниц и других ресурсов между клиентом и сервером.

---

# 2. Концепция Web (1989)

Концепция Web включает четыре основных компонента:

1. **HTML** — язык гипертекстовой разметки страниц  
2. **HTTP** — протокол передачи гипертекстовых страниц  
3. **Web-сервер** — программа, которая хранит и отправляет страницы  
4. **Web-браузер** — программа для просмотра страниц  

---

# 3. Гипертекст и HTML

+ HTML используется для создания веб-страниц.

Пример простой HTML-страницы:

<html>
<h1>Протокол HTTP</h1>
</html>

> HTML позволяет создавать гиперссылки, которые связывают страницы между собой.

4. URL

URL (Uniform Resource Locator) — единообразный указатель местонахождения ресурса.

Используется в гиперссылках для доступа к ресурсам в интернете.

Пример URL:

http://www.site.com/courses/networks

Структура URL:

Часть	Описание
http://	протокол
www.site.com	адрес сервера
/courses/networks	путь к странице

URL используется не только для гипертекста, но и для других типов файлов.

5. Другие протоколы

Помимо HTTP существуют:

https:// — защищённая версия HTTP

ftp:// — протокол передачи файлов

6. Версии протокола HTTP
Версия	Год	Особенности
HTTP 1.0	1996	первая официальная версия
HTTP 1.1	1997	кэширование, постоянные соединения, аутентификация
HTTP/2	2015	улучшенная безопасность и производительность
HTTP/3	2022	современная версия, внедряется
7. Принцип работы HTTP

HTTP работает по модели клиент-сервер.

Клиент (браузер) отправляет запрос

Сервер отправляет ответ

На транспортном уровне используется протокол TCP, стандартный порт:

80
8. Режим работы HTTP
Версия	Тип передачи
HTTP 1.1 и ранее	текстовый
HTTP/2, HTTP/3	бинарный

Текстовый режим означает, что серверы передают обычные текстовые строки.

9. Структура HTTP-запроса

HTTP-запрос состоит из трёх частей:

Строка запроса

GET /courses HTTP/1.1

Заголовки (необязательно)

Host: site.com

Для HTTP/1.1 заголовок Host обязателен.

Тело сообщения (необязательно)
Используется для передачи данных пользователя.

10. Методы HTTP
Метод	Назначение
GET	запрос веб-страницы или ресурса
POST	передача данных на сервер
HEAD	запрос только заголовков
PUT	загрузка страницы на сервер
DELETE	удаление ресурса
OPTIONS	запрос поддерживаемых методов
11. Коды состояния HTTP

Ответ сервера содержит код состояния.

1XX — информационные

Сервер сообщает дополнительную информацию.

2XX — успешное выполнение

Пример:

200 OK
3XX — перенаправление
Код	Значение
301	постоянное перенаправление
302	временное перенаправление
4XX — ошибка клиента
Код	Значение
403	доступ запрещён
404	страница не найдена
5XX — ошибка сервера
Код	Значение
500	внутренняя ошибка сервера
501	метод не реализован

14. Современный Web

Современные веб-приложения значительно сложнее:

страницы содержат множество элементов

используются веб-приложения вместо статических страниц

применяются HTTP/2 и HTTP/3 для повышения производительности

15. Использование HTTP в API

HTTP используется не только для веб-страниц, но и для API.

API позволяет:

вызывать программы через HTTP

передавать данные между сервисами

создавать веб-приложения и мобильные приложения
