#  Существуют два типа ссылок на файлы:
---
- **Жёсткие ссылки (hard links)**
> *Hard link — это второе имя одного и того же файла.*

- **Символические ссылки (symbolic links, symlinks)**
> *Symlink — это файл-указатель на другой файл.*

+ Они позволяют создавать дополнительные «имена» для файлов и каталогов, но работают по-разному.
---
## 1. Жёсткие ссылки (Hard Links)
+ Жёсткая ссылка — это дополнительное имя для существующего файла.  
+ Фактически это ещё одна запись в файловой системе, указывающая на тот же inode.
## Особенности:
+ Данные на диске не копируются
+ У файла увеличивается счётчик ссылок
+ Можно создавать только для файлов
+ Нельзя создавать для каталогов (обычным пользователям)
+ Нельзя создавать между разными файловыми системами
+ Удаление одного имени не удаляет данные, пока есть другие ссылки
### Создание:
`ln original.txt symlink.txt`
+ Использовать жёсткие ссылки, если:
  *Нужно несколько имён одного файла*
  *Важно сохранение данных при удалении одного имени*
---
## 2. Символические ссылки (Symbolic Links):
+ Символическая ссылка — это специальный файл, который содержит путь к другому файлу или каталогу.
+ Работает как «ярлык» в Windows.
## Особенности:
+ Symlink хранит путь к файлу
+ Имеет собственный inode
+ Может ссылаться на файл или каталог
+ Может указывать на файл в другой файловой системе
+ Проверить:
`ls -l`
### Создание:
`ln -s original.txt symlink.txt`
+ Использовать символические ссылки, если:
  *Нужно ссылаться на каталог*
  *Нужно ссылаться между разными разделами*
  *Требуется гибкость и понятная структура проекта*
