# `Linux.Commands`
---
## Directory Managment
+ **ls - просмотр директории ( -l - показывает информацию о правах, времени и символам);**
+ **cd - переход по директориям ( . . - переход в предыдущую директорию );**
+ **pwd - просмотр пути пользователя;** 
---
## Useful Utilities
+ **clear - очистка терминала;**
+ **history - история запросов в терминале;**
> history -c - очистка истории;
---
+ **man - помощь по информации о команде.**
+ **help - помощь на более сжатой информации.**
+ **q - выход из заданого файла;** 
> ( ctrl + c - завершает операцию )
---
+ <u>(word + * - показывает количество выбранных word файлов );</u>
+ <u>(word + [ value ] - показывает определенное количество word исходя из заданного value );</u>
+ <u>( command ) && ( command ) - если 1-ая команда произошла успешно, совершить 2-ую команду; </u>
+ <u>( command ) || ( command ) - если 1-ая команда была отказана, совершить 2-ую команду; </u>
---
## Text Managment 
+ **cat - просмотр заданного файла;**
> (  cat >txt - можно создать файл без "touch" и редактировать его;
> cat txt >txt.2 - переносит данные первого и стирает данные второго файла;
> cat txt >>txt.2 - переносит данные первого файла и сохраняет предыдущую информацию;  )
+ cat .txt | head (+ значение) - покажет первые n-кол во файлов
+ tail (+ значение) - покажет последние n-кол во файлов
+ **less - просмотр без редактирования;**
> (+ / (просмотр по ключевым словам) )
+ echo "word" - выводит информацию при совершении операции.
+ grep - нахождение вхождения значения и выводит значение;
> ( grep (word) < (название файла) - выведет ворд, который находится в данном файле )
---
## File Managment
+ touch - создание файла.
> ( touch -t ( time ) txt  - изменит выбранное время файла )
> ( touch -r txt txt.2 - переносит мета-данные из txt в txt.2 )
+ rm -r +файл - удаление файла/папки/ветви рекурсивно.
+ rm -rf +файл - удаление файла/папки/ветви принудительно.
+ cp txt txt.2 - копировать файл.
> cp -r txt txt.2 - рекурсивно копируется  дериктория txt в txt.2
> cp -r txt/* txt2 - рекурсивно копируется содержимое папки txt в txt.2
> cp -u - не понял, но что-то связано со штампом времени;
+ mv - переместить файл.
> (+и заменяет содержимое уже созданного до этого файла) 
+ mkdir - создание дериктории;
> ( mkdir -p (/dir1/docs/resume) - создание ветвей дериктории );
+ rmdir - удаление дериктории
> (rmdir -p /+путь - удаление ветви дериктории);
